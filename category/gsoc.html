Things have been done

In the first 4 days of GSoC, I successfully ported my existing code into a forked biopython distribution in my github. I didn't pay much effort to the code organization as all of them are put into __init__.py. Maybe when the code grows larger I need to make them more organized. The main class for codon alignment is CodonAlignment class that inherits from MultipleSeqAlignment. The main difference is the __getitem__() method, which in CodonAlignment is codon slice (i.e. three nucleotides per slice, see below). I also changed the __str__() method so that it is more suitable to print a CodonAlignment. Another thing I've done is the build method, which accept a protein MultipleSeqAlignment object and a set of SeqRecord of its corresponding nucleotide sequences to come up with a CodonAlignment. The function now will translate the protein sequences into a long regular expression based on the user specified CodonTable in biopython. It will then use the regualr expression to scan the nucleotide sequence to find a match. No mismatch and frameshift is allowed right now in this step. Finally, a CodonAlignment will be returned to the user for further manipulation.

Some Examples

It is best to illustrate CodonAlignment using simple examples. Here are some (assuming init.py and test_data/ are in your current python directory):

  1. Build a CodonAlignment from Protein Alignment and Nucleotide Sequences. In the following example, nucl_seq1 is an iterator returned by SeqIO.parse(). I assume the order of sequence records stored in prot_aln1 and nucl_seq1 are the same. ```python >>> from Bio import SeqIO, AlignIO >>> from Bio.Alphabet import IUPAC >>> from init import build, default_alphabet >>> prot_aln1 = AlignIO.read('test_data/pro1.aln', 'clustal', alphabet=IUPAC.protein) >>> nucl_seq1 = SeqIO.parse('test_data/nucl1.fa', 'fasta', alphabet=default_alphabet) >>> codon_aln1 = build(prot_aln1, nucl_seq1) >>> print codon_aln1 CodonAlignment Object HasStopCodon(Gapped(IUPACUnambiguousDNA(), '-'), '*') alignment with 3 rows and 555 columns (185 codons) AGAGGCGATCAACGCAGCAACTTCCAGCTGTCTCCCTCCACCATGCAGATC...GGGGCATGC isotig69710 ------------------------------------------ATGAAAGTC...--------- ENSG00000108691:ENST00000580907 ------------------------------------------ATGAAAGTC...--------- ENSG00000108691:ENST00000225831

    ```

  2. It is also possible to use SeqRecord dict read by SeqIO.index() to build CodonAlignment. In this case, I assume nucleotide SeqRecord and protein SeqRecord sharing the same id are a match. e.g. ```python >>> prot_aln2 = AlignIO.read('test_data/pro3.aln', 'clustal', alphabet=IUPAC.protein) >>> nucl_seq2 = SeqIO.index('test_data/nucl3.fa', 'fasta', alphabet=default_alphabet) >>> codon_aln2 = build(prot_aln2, nucl_seq2) >>> print codon_aln2 CodonAlignment Object HasStopCodon(Gapped(IUPACUnambiguousDNA(), '-'), '*') alignment with 10 rows and 3366 columns (1122 codons) ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000300134 ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000543873 ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000556155 ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000454075 ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000538913 ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000537215 ---------------------------------------------------...CCCAGTTGG ENSG00000166888:ENST00000535201 GAGGTTACTCAGAGTAGGAGGAAGCCGGTCAGAGAGGGCAGACCCTGGGAA...CTGCTTTGC isotig46679 ------------GCTAGGAGA------------------------------...--------- isotig12565 ------------GCTAGGAGA------------------------------...--------- isotig12566

    ``` 3. Then let's look at the SeqRecord in CodonAlignment. The CodonSeq object stored in it is a bit different from normal Seq Object. The main difference it that it is sliced by codon not nucleotide. Let's see the following example.

    ```python

    codon_record = codon_aln1[0] print codon_record SeqRecord(seq=CodonSeq('AGAGGCGATCAACGCAGCAACTTCCAGCTGTCTCCCTCCACCATGCAGATCTCC...TGC', Gapped(HasStopCodon(IUPACUnambiguousDNA(), ''), '-')), id='isotig69710', name='', description='', dbxrefs=[]) codon_record.seq CodonSeq('AGAGGCGATCAACGCAGCAACTTCCAGCTGTCTCCCTCCACCATGCAGATCTCC...TGC', Gapped(HasStopCodon(IUPACUnambiguousDNA(), ''), '-')) print codon_record.seq.class.name CodonSeq print codon_record.seq[0] AGA print codon_record.seq[:3] AGAGGCGAT print codon_record.seq[-3:] GGGGCATGC codon_record.seq.get_codon_num() 185

    ```

This example above is the basic usage of CodonAlignment. If you have any suggestions or request some features are useful for codon alignment please let me know.
