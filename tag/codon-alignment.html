Fri 28 June 2013
By Zheng Ruan

In GSoC.

tags: Codon AlignmentGSoC

Things have been done

This week I finished the mismatch support for codon alignment construction. The underlying algorithms is very similar to that implemented in pal2nal. Specifically, if whole protein translated codon re failed to find a hit in the nucleotide sequences, small 10 amino acids anchors are used to scan it individually. Unmatched anchors are represented as '...' * anchor_len in the re. The program then use the concatenated anchor expression to scan the whole nucleotide sequence. If this time, a match is found, it will read through the nucleotide sequence according to the match span and figure out which positions are not matched. Warning will raise up for unmatched positions. Another thing I've done is adding the corr_dict option for AlignIO.build method. This optional argument will accept a dict object that tells the id correspondance between protein sequence records and nucleotide records. For the IO support, because of the codon slice in CodonSeq, CodonAlignment will not be correctly written using normal AlignIO.write. Therefore, I wrote a toMultipleSeqAlignment() method in CodonAlignment. As its name indicated, it will return a MultipleSeqAlignment object with Seq to store sequences. The user can the write out their codon alignment into any format supported by AlignIO. Finally, I also set up a test for CodonAlign module. It is named test_CodonAlignment in Tests directory. Some warnings will raise up because the example sequences contains mismatches.

Some Examples

I will still write some examples here to illustrate the function I've implemented. I assume __init__.py and test_data are in your current python directory:

1. Build a CodonAlignment from Protein Alignment and Nucleotide Sequences when some of the sites are incompatible.

>>> from Bio.Alphabet import IUPAC
>>> from __init__ import build, default_codon_alphabet
>>> nucls5 = SeqIO.parse('test_data/nucl5.fa', 'fasta', alphabet=IUPAC.IUPACUnambiguousDNA())
>>> pro_alignr5 = AlignIO.read('test_data/pro5.aln', 'clustal', alphabet=IUPAC.protein)
>>> aln = build(pro_align5, nucls5, alphabet=default_codon_alphabet)

  1. To write out a constructed codon alignment, you need to first convert it into a MultipleSeqAlignment object and then use the functions in AlignIO to write out. Here is an example:
>>> AlignIO.write(codonaln, 'ex.clw', 'clustal')
# If you open the ex.clw file, it is not properly formated.
# This is caused by the codon slice of CodonSeq objects.
# Here is the correct way to write it out.
>>> aln = codonaln.toMultipleSeqAlignment()
>>> AlignIO.write(aln, 'ex1.clw', 'clustal')
# This time it will look much better
