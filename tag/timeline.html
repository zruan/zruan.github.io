Project Timeline

Mon 24 June 2013
By Zheng Ruan

In Timeline.

tags: Codon AlignmentGSoCTimelinePlan

This timeline is set up according to Karen Cranston's suggestion. I will update this page according to the progress of my current coding status and suggestions from others. My original proposal also contains a draft timeline, which I will still generally follow. It can be found at Melange. Hope this can divide major difficulties into small pieces and help me successfully finish the project!

Detailed Timeline

Week 1

(June 17 -- June 23)

  • Goal Set up the framework of codon alignment in biopython
  • Tasks
    1. Write a CodonAlignment class that inherits from MultipleSeqAlignment class of Biopython;
    2. Adapt my existing CodonAlign functionality into CodonAlignment object;
    3. Replace my own codon table with Biopython codon tables;
    4. Wisely match DNA sequences and protein sequences while constructing codon alignment;
    5. Speed up the existing implementation.

All the 5 items listed above have been finished last week. In terms of item 5, I hardly pays any effort as I assume the protein sequences can be exactly translated by DNA sequences. The speed is more of a concern if I am dealing with mismatch and frameshift while building the alignment. Therefore, this will be a long term task (at least for several weeks). Another things I need to mention is that I've written a CodonSeq class to store sequences in CodonAlignment. Peter seemed not very favor of such a concept. I'm not sure if I need to return to the Seq class since codon sequences have much unique properties compared with normal Seq. I also implemented a codon alphabet to check the CodonSeq sequences, which is not in the original plan.

Week 2

(June 24 -- June 30)

  • Goal Support mismatches between DNA sequences and AA alignment
  • Tasks
    1. Write a function to find the discrepancies between a given protein sequence and coding DNA sequence (Check all reading frames);
    2. Add mismatch tolerance option when build codon alignment (this involves scoring);
    3. If failed to build codon alignment in any way, raise an error indicating differences.
    4. If SeqRecord contain annotation, try to use that information to build alignment.
    5. Speed up the existing implementation.

Week 3

(July 1 -- July 7)

  • Goal Support frameshift of DNA translation
  • Tasks
    1. Write a function to find the frameshift between a given protein sequence and coding DNA sequence;
    2. Add frameshift tolerance option when build codon alignment (this also involves scoring);
    3. If failed to build codon alignment in any way, raise an error indicating differences.
    4. If SeqRecord contain annotation, try to use that information to build alignment.

Week 4

(July 8 -- July 17)

  • Goal Make codon alignment functionality capable of dealing with both mismatches and frameshift
  • Tasks
    1. Write a function to unify mismatch detection and frameshift detection, whichever is more likely;
    2. Unify mismatch-frameshift tolerance option when build codon alignment (this involves scoring);
    3. If failed to build codon alignment in any way, raise an error indicating differences.

Plans for Following Weeks will Come Soon

Any suggestions are welcome. For the project source code, please visit my codonAlign github repo.
