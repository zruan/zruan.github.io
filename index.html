Project Timeline

Fri 14 June 2013
By Zheng Ruan

In GSoC.

tags: Codon AlignmentGSoCTimeline

This timeline is set up according to Karen Cranston's suggestion. I will update this page according to the progress of my current coding status. My original proposal also contains a draft timeline, which I will still generally follow. It can be found at Melange.

Detailed Timeline

Week 1

(June 17 -- June 23)

  • Goal Set up the framework of codon alignment in biopython
  • Tasks
    1. Write a CodonAlignment object that inherits from MultipleSeqAlignment Object of Biopython;
    2. Adapt my existing CodonAlign functionality into CodonAlignment object;
    3. Replace my own codon table with Biopython codon tables;
    4. Wisely match DNA sequences and protein sequences while constructing codon alignment;
    5. Speed up the existing implementation.

Week 2

(June 24 -- June 30)

  • Goal Support mismatches between DNA sequences and AA alignment
  • Tasks
    1. Write a function to find the discrepancies between a given protein sequence and coding DNA sequence (Check all reading frames);
    2. Add mismatch tolerance option when build codon alignment (this involves scoring);
    3. If failed to build codon alignment in any way, raise an error indicating differences.
    4. If SeqRecord contain annotation, try to use that information to build alignment.

Week 3

(July 1 -- July 7)

  • Goal Support frameshift of DNA translation
  • Tasks
    1. Write a function to find the frameshift between a given protein sequence and coding DNA sequence;
    2. Add frameshift tolerance option when build codon alignment (this also involves scoring);
    3. If failed to build codon alignment in any way, raise an error indicating differences.
    4. If SeqRecord contain annotation, try to use that information to build alignment.

Week 4

(July 8 -- July 17)

  • Goal Make codon alignment functionality capable of dealing with both mismatches and frameshift
  • Tasks
    1. Write a function to unify mismatch detection and frameshift detection, whichever is more likely;
    2. Unify mismatch-frameshift tolerance option when build codon alignment (this involves scoring);
    3. If failed to build codon alignment in any way, raise an error indicating differences.

Plans for Following Weeks will Come Soon

Any suggestions are welcome. For the project source code, please visit my codonAlign github repo.
