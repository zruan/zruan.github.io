7th Diary

Mon 19 August 2013
By Zheng Ruan

In GSoC.

tags: Codon AlignmentGSoC

Things have been done

The job I've mainly done since last week is maximum likelihood method for dN/dS ratio estimation. It is one of the most difficult part of my GSoC project. Now I have implemented all the basic steps described in Yang's 2000 MBE paper (PMID: 10666704). The most computational intensive step is to calculate the Q matrix, which involves matrix exponential. I employed scipy.linalg.expm to achieve the task. For the constraint optimization, I used the L-BFGS-B method in scipy.optimize.minimize and it has reasonable performance (about several seconds to have pairwise dN/dS estimated). The result is not exactly the same with the example in Yang's book but very similar, so I expect the difference might be caused by slightly different sequences. Maximum Likelihood method is a model based method, which is able to incorporate different substitution schemes. In the next week, I will allow user to specify different substitution model. While I am writing documentation for Bio.CodonAlign, several bugs has been found and they are fixed when I found them. Therefore, I am writing documentation in a rather slow pace. I also changed the upper length limit of forward frameshift events. The program will now allow more than 2 bases of forward frameshift but it should not exceed the anchor_len . If the purpose is to handle intron in DNA sequence, this is not useful as intron is typically very large. I need to think about it more carefully.

I don't have examples this week, as ML method is used exactly the same as the others.

Future Plan

  1. Implement YN00 method
  2. Add more options for ML method (allow different substitution model)
  3. Continue Writing documentation for codon alignment construction.
