By Zheng Ruan

In GSoC.

tags: Codon AlignmentGSoC

Things have been done

In the last coding week for my GSoC project, I am mainly dealing with issues left from previous weeks as well as several new features. First, a cfreq option has been added to the maximum likelihood method and it allows the user to specify equilibrium codon frequency model (available options include F1x4, F3x4 and F61). Second new feature is dN, dS tree reconstruction. This is done by using Yanbo Ye's Code (specifically, DistanceMatrix and DistanceTreeConstructor classes). This method is implemented in CodonAlignment class so that user can build a dN tree and dS tree once it get the alignment. Furthermore, I corrected all the things Eric pointed out in my code several days ago. toCodonAlignment is now a classmethod for called from_msa in CodonAlignment class, rather than an independent function. I also speed up the Mcdonald Kreitman test implementation by breaking the _dijkstra() once it arrives at the target codon. However, I think I can make it even faster by building codon to codon substitution matrix on the fly (I may do this after my GSoC). In addition, I removes the Scipy dependency by using the chisq.py from Eric's biofrills/stats/. When the convergency problem in Phylo/PAML/chi2.py is solved, I may shift it back to Biopython's chi2 implementation. I also re-organized the cal_dn_ds method so that it is more readable.

Some Examples

I will illustrate the two new features described above.

  1. Using different equilibrium codon frequency model to estimate dN, dS under ML.
>>> from Bio import SeqIO, AlignIO
>>> from Bio import CodonAlign
>>> from Bio.Data.CodonTable import generic_by_id
>>> from Bio.Alphabet import IUPAC
>>> from Bio.CodonAlign.CodonSeq import cal_dn_ds

>>> aln = AlignIO.read('pro.aln', 'clustal', alphabet=IUPAC.protein)
>>> seq1 = SeqIO.read('rbcL_cucu_mRNA.fa', 'fasta', alphabet=IUPAC.IUPACUnambiguousDNA())
>>> seq2 = SeqIO.read('rbcL_nic_mRNA.fa', 'fasta', alphabet=IUPAC.IUPACUnambiguousDNA())

>>> codon_aln = CodonAlign.build(aln, [seq1, seq2], codon_table=generic_by_id[11])
>>> dN, dS = cal_dn_ds(codon_aln[0], codon_aln[1], method='ML', cfreq='F1x4')
>>> print dN, dS
0.0345408062877 0.314510886694
>>> dN, dS = cal_dn_ds(codon_aln[0], codon_aln[1], method='ML', cfreq='F3x4')
>>> print dN, dS
0.0342472820368 0.420673537297
>>> dN, dS = cal_dn_ds(codon_aln[0], codon_aln[1], method='ML', cfreq='F61')
>>> print dN, dS
0.0355901166208 0.44417445805

By default, ML method will use F3x4 to estimate dN and dS.
